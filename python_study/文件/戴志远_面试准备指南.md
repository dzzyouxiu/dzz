### 问题2：如何处理办公网络故障？

**回答示例**：

```
处理网络故障时，我遵循以下流程：
1. **定位问题范围**：确认是单个用户还是大面积故障
2. **分层排查**：
   - 检查客户端（网卡状态、IP配置）
   - 检查接入层（交换机端口、AP信号）
   - 检查核心设备（路由状态、带宽使用）
3. **常见解决方案**：
   - IP冲突：释放并重新获取IP
   - 端口故障：切换端口或重启设备
   - 带宽占用：定位占用源并优化

例如，去年处理过一次核心交换机故障，我立即启用备用链路，2小时内恢复全部业务。
```

```

1. **策略中心**：配置设备合规策略（系统版本要求、屏幕锁、加密）
2. **数据保护**：设置敏感文件外发规则，配置外发预警
3. **权限管理**：针对不同人群设置差异化权限，预离职人员自动收回权限
4. **合规审计**：定期审计操作日志，确保策略执行合规
```
