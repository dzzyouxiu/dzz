## 二、面试问答

### Q1：请描述飞书访问控制策略的配置逻辑，如何实现基于设备的管理？

**回答框架：条件判断 → 配置路径 → 实际效果**

飞书的访问控制策略核心是通过**条件组合**来控制成员访问企业数据的权限。

**配置路径：**

1. 进入「管理后台 → 策略中心 → 访问控制」
2. 新建策略，配置条件维度：
   - **设备归属**：公司设备/个人设备（通过设备管理模块绑定）
   - **设备状态**：是否在线、是否启用设备锁
   - **客户端类型**：iOS/Android/Windows/Mac/Web
   - **网络环境**：可配置IP围栏，限制特定IP段或排除特定IP

**实际场景举例：**
比如要求"只有公司设备+Windows客户端+内网IP才能访问飞书文档"，我会配置三个条件为"与"关系。这样个人设备或外部网络环境会被直接拦截访问。

**注意事项：**

- 策略优先级按列表顺序从上到下匹配，命中即执行
- 可以设置"满足条件允许/拒绝"和"不满足条件允许/拒绝"四种动作
- 建议先在测试部门验证，再全量发布

### Q2：数据防泄漏（DLP）策略是怎么配置的？覆盖哪些场景？

**回答框架：场景识别 → 规则配置 → 处置动作**

DLP是飞书数据安全的核心，我主要在**策略中心 → 数据防泄漏**模块配置。

**覆盖场景：**

1. **即时消息**：聊天记录中的敏感信息识别
2. **云文档**：文档内容、评论、分享链接

**配置步骤：**

1. 新建DLP策略，选择应用范围（全员/部门/特定用户组）
2. 配置识别规则，支持三种方式：
   - **关键词**：如"机密""绝密""薪资""合同"
   - **正则匹配**：如`\d{15,18}`（身份证号）、`\$[\d,]+`（金额）
   - **敏感信息类型**：内置模板包括身份证、银行卡、手机号、邮箱等
3. 设置处置动作：
   - **拦截**：直接阻止发送/分享
   - **审计**：放行但记录日志，供后续追溯
   - **关闭对外分享**：自动取消已分享链接的外部访问权限

**实际效果：**
我们上线DLP后，即时消息场景拦截了12起包含客户信息的敏感外发，云文档场景关闭了8个异常分享链接。

### Q3：密级标签体系如何设计？调低密级为什么要走审批？

**回答框架：分级标准 → 标签配置 → 审批流设计**

**密级标签设计（L1/L2/L3三级）：**

- **L1（公开）**：对外宣传资料、市场营销内容
- **L2（内部）**：内部通知、工作流程、的一般文档
- **L3（机密）**：薪酬数据、客户名单、核心技术文档、合同协议

**配置路径：**

1. 「管理后台 → 策略中心 → 密级标签」创建标签体系
2. 绑定标签到人员或文档
3. 发布标签策略，配置强制覆盖范围

**调低密级走审批的原因：**
密级调整是数据泄露的高风险操作。比如L3机密文档被随意降级为L1公开，就会造成信息外泄。我们配置的是：

- 调低密级必须走审批流
- 审批人设为直属上级 + 安全管理员双重审批
- 保留完整的审批记录日志

### Q4：水印体系如何配置？透明水印和明水印有什么区别？

**回答框架：两种水印 → 配置路径 → 溯源能力对比**

**明水印 vs 透明水印：**

| 类型   | 可见性   | 用途   | 溯源能力         |
| ---- | ----- | ---- | ------------ |
| 明水印  | 肉眼可见  | 震慑作用 | 直接显示身份信息     |
| 透明水印 | 肉眼不可见 | 溯源追踪 | 通过技术手段提取用户ID |

**配置路径（管理后台 → 数据保护 → 水印设置）：**

1. **明水印**：设置显示内容（姓名、手机号后四位、工号），可自定义水印密度和颜色
2. **透明水印**：开启后，用户截图时可提取用户身份信息
3. **覆盖范围**：
   - 线上页面水印（文档阅读页、消息预览）
   - 下载文件水印（导出PDF、表格）
   - 截图水印浮层（截图时自动叠加）

**实际场景：**
曾有员工试图通过截图泄露客户名单，透明水印记录了截图时间戳和账号ID，直接定位到人。

### Q5：什么是BYOK？加密密钥是如何管理的？

**回答框架：概念解释 → 加密算法 → 迁移步骤**

**BYOK（Bring Your Own Key）概念：**
企业可使用自己的密钥管理系统来保护飞书数据，而非使用飞书默认密钥。2026年起，飞书将强制开启自主密钥配置。

**加密配置：**

- 支持算法：AES\_256\_GCM（国际标准）和SM4（国密标准）
- 密钥生命周期管理：创建 → 启用 → 轮换 → 禁用

**迁移步骤：**

1. 在密钥管理平台生成密钥材料
2. 「管理后台 → 数据保护 → 加密设置」导入密钥
3. 配置密钥启用策略，选择加密范围（全员/指定部门）
4. 验证加密生效（新建文档测试是否使用新密钥）

**注意事项：**

- 密钥泄露 = 数据泄露，必须妥善保管
- 建议开启密钥轮换（如每年自动更新）
- 越南本地化部署需注意SM4合规要求

### Q6：飞书权限体系是如何设计的？超级管理员和自定义管理员有什么区别？

**回答框架：角色分层 → 权限对比 → 分权原则**

**管理员角色：**

| 角色     | 权限范围            | 适用场景       |
| ------ | --------------- | ---------- |
| 超级管理员  | 全企业数据，可管理所有管理员  | 创始人/一把手    |
| 自定义管理员 | 按数据范围隔离，仅管理指定范围 | 部门负责人/IT运维 |

**数据范围隔离配置：**
「管理后台 → 管理员 → 数据管理范围」可设置：

- 部门范围：仅能看到/管理指定部门
- 用户组范围：仅管理特定用户组
- 应用范围：仅管理特定应用
- 会议室范围：仅管理特定会议室资源

**分权原则：**

- 最小权限原则：管理员只给必要权限
- 职责分离：安全管理与日常运营分岗
- 我们配置了3名自定义管理员，分别负责研发、越南本地团队、行政三个独立范围，互不干扰

### Q7：文档权限默认值的优先级规则是什么？如何处理权限冲突？

**回答框架：优先级列表 → 冲突场景 → 处理原则**

**权限优先级（从高到低）：**

1. **成员**：直接针对某个成员设置的权限最高
2. **用户组**：加入用户组的成员继承用户组权限
3. **子部门**：子部门的权限优于父部门
4. **上级部门**：逐级向上直到全局
5. **全局**：默认全局设置兜底

**冲突处理原则：**

- **就高原则**：多个权限源叠加时，取最高权限
- **禁止优于允许**：如果任一权限源设置"禁止"，则禁止生效
- 例：成员A同时属于"财务组"和"审计组"，财务组禁止外分享，审计组允许外分享，最终A无法外分享

**实际操作：**
处理过研发部门成员同时属于"外包人员"用户组的冲突，配置规则为：外包人员用户组权限最低，研发部门成员权限优先。

### Q9：员工离职时，如何通过飞书后台完成全流程安全处理？

**回答框架：账号处置 → 权限回收 → 数据审计 → 收尾确认**

**标准离职流程：**

1. **账号状态变更**
   - 「管理后台 → 成员与部门 → 成员管理」找到离职员工
   - 操作：禁用账号或转移账号数据
2. **权限即时回收**
   - 撤销所有管理员权限
   - 取消应用授权
   - 移除所有用户组
   - 回收共享文档权限（批量转移文档所有权给直属上级）
3. **数据安全审计**
   - 查询该员工180天行为日志
   - 重点审查：异常登录时间、敏感文档访问记录、DLP命中事件
   - 检查是否有大文件外发或异常分享行为
4. **沟通权限管控**
   - 「管理后台 → 权限与安全 → 对外沟通权限」设置黑名单
   - 禁止该员工与其他成员私信（防止离职前信息转移）
5. **收尾确认**
   - 完成以上步骤后，注销账号
   - 保留日志记录至少180天供追溯

### Q10：发现数据泄露后，应急响应流程是什么？

**回答框架：发现 → 遏制 → 溯源 → 修复 → 复盘**

**应急响应五步法：**

1. **发现与上报（黄金1小时）**
   - 触发来源：DLP命中告警、员工举报、安全监控告警
   - 第一动作：确认泄露范围，截图/录屏保留证据
2. **遏制扩散**
   - 立即关闭泄露数据的分享链接（管理后台 → 云文档 → 分享设置）
   - 如果是内部人员泄露：立即禁用该员工账号
   - 如果是外部攻击：切换访问控制策略，限制可疑IP
3. **溯源分析**
   - 调取审计日志：
     - 登录日志：查找异常登录IP/设备
     - DLP日志：定位触发时间点和内容
     - 文档操作日志：查看是否被下载/转发
   - 结合水印记录：如果是截图泄露，可提取透明水印溯源
4. **修复闭环**
   - 通知受影响方（如客户泄露需通知客户）
   - 评估数据恢复可能性
   - 更新安全策略，补上漏洞
5. **复盘与预防**
   - 输出事件报告（时间线、影响范围、根因）
   - 完善安全策略（如补充DLP规则、加强权限管控）
   - 组织安全培训，避免再次发生
