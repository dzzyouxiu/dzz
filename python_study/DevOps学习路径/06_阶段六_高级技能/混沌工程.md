# 混沌工程

## 混沌工程简介

### 什么是混沌工程
- 在分布式系统上进行实验性演练
- 建立系统抵御生产环境动荡的信心
- Netflix首创（Chaos Monkey）

### 混沌工程原则

```
1. 建立稳定状态假说
2. 多样化真实世界事件
3. 在生产环境运行实验
4. 持续自动化运行
5. 最小化爆炸半径
```

### 实验流程

```
提出假设 → 选择事件 → 定义范围 → 
运行实验 → 验证假设 → 学习改进
```

| 类型 | 说明 | 工具 |

## 网络故障注入

### 使用Linux TC（Traffic Control）

```bash
# 添加网络延迟（500ms）
sudo tc qdisc add dev eth0 root netem delay 500ms

# 添加网络延迟和抖动（500ms ± 50ms）
sudo tc qdisc add dev eth0 root netem delay 500ms 50ms

# 添加丢包（10%）
sudo tc qdisc add dev eth0 root netem loss 10%

# 添加数据包损坏
sudo tc qdisc add dev eth0 root netem corrupt 5%

# 添加数据包重复
sudo tc qdisc add dev eth0 root netem duplicate 5%

# 组合多个效果
sudo tc qdisc add dev eth0 root netem delay 200ms loss 5%

# 查看当前规则
sudo tc qdisc show dev eth0

# 删除规则
sudo tc qdisc del dev eth0 root netem

# 修改规则
sudo tc qdisc change dev eth0 root netem delay 100ms
```

### 使用Toxiproxy

```bash
# Docker运行
docker run -d -p 8474:8474 -p 19200:19200 shopify/toxiproxy

# 或下载二进制
wget https://github.com/Shopify/toxiproxy/releases/download/v2.5.0/toxiproxy-server-linux-amd64
chmod +x toxiproxy-server-linux-amd64
./toxiproxy-server-linux-amd64 &

# 安装CLI
wget https://github.com/Shopify/toxiproxy/releases/download/v2.5.0/toxiproxy-cli-linux-amd64
chmod +x toxiproxy-cli-linux-amd64
sudo mv toxiproxy-cli-linux-amd64 /usr/local/bin/toxiproxy-cli

# 创建代理
toxiproxy-cli create mysql -l localhost:23306 -u localhost:3306

# 查看代理列表
toxiproxy-cli list

# 添加延迟
toxiproxy-cli toxic add mysql -t latency -a latency=1000

# 添加丢包
toxiproxy-cli toxic add mysql -t limit_data -a rate=1000

# 移除toxic
toxiproxy-cli toxic remove mysql -n latency_downstream

# 删除代理
toxiproxy-cli delete mysql
```

## 资源耗尽

### stress-ng

```bash
# 安装
sudo apt install stress-ng

# CPU压力（4个worker）
stress-ng --cpu 4 --timeout 60s

# 内存压力（分配2GB）
stress-ng --vm 2 --vm-bytes 1G --timeout 60s

# 磁盘I/O压力
stress-ng --hdd 2 --hdd-bytes 1G --timeout 60s

# 组合压力
stress-ng --cpu 2 --vm 1 --vm-bytes 512M --timeout 2m

# 详细输出
stress-ng --cpu 4 -v --timeout 60s

# 查看系统状态
watch -n 1 'uptime && free -h'
```

### stress

```bash
# 安装
sudo apt install stress

# CPU压力（4个进程）
stress --cpu 4 --timeout 60

# 内存压力
stress --vm 2 --vm-bytes 256M --timeout 60

# I/O压力
stress --io 4 --timeout 60

# 组合
stress --cpu 2 --io 2 --vm 1 --vm-bytes 128M --timeout 60s
```

  name: network-delay

      - default

## HTTP服务故障

### 使用Python脚本

```python
#!/usr/bin/env python3
"""简单的故障注入脚本"""

import random
import time
import requests

def inject_fault(url, fault_type):
    """注入故障"""
    if fault_type == "delay":
        time.sleep(random.randint(1, 5))
    
    try:
        response = requests.get(url, timeout=10)
        print(f"Response: {response.status_code}")
    except Exception as e:
        print(f"Error: {e}")

def kill_process(name):
    """杀死进程"""
    import subprocess
    subprocess.run(["pkill", "-f", name])

if __name__ == "__main__":
    # 随机注入故障
    if random.random() < 0.3:  # 30%概率
        print("Injecting fault...")
        kill_process("nginx")
```

## 混沌工程最佳实践

### 1. 从简单开始

```
1. 先在测试环境实验
2. 从小规模故障开始
3. 逐步增加复杂度
```

### 2. 定义稳态

```
- 健康检查端点返回200
- 响应时间<500ms
- 错误率<1%
```

### 3. 最小化爆炸半径

```
- 只影响部分实例
- 设置实验持续时间
- 有自动回滚机制
```

### 4. 自动化

```
- 集成到CI/CD
- 定期运行实验
- 自动收集结果
```

### 5. 记录和学习

```
- 记录所有实验
- 分析失败原因
- 更新系统设计
```
