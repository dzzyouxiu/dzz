# 网络配置问题解决方案

## 问题概述

在Ubuntu系统中使用Netplan进行网络配置时，可能会遇到以下错误：

1. **权限错误**：`Permissions for /etc/netplan/00-installer-config.yaml are too open`
2. **语法过时**：`gateway4 has been deprecated, use default routes instead`
3. **YAML语法错误**：`Invalid YAML: inconsistent indentation`
4. **服务问题**：`systemd-networkd is not running`

## 解决方案

### 1. 修复文件权限

**问题**：配置文件权限过于开放，存在安全风险

**解决方法**：
```bash
sudo chmod 600 /etc/netplan/00-installer-config.yaml
```

### 2. 修复YAML缩进问题

**问题**：YAML文件缩进不一致，导致解析失败

**解决方法**：
1. 编辑配置文件：
   ```bash
   sudo nano /etc/netplan/00-installer-config.yaml
   ```

2. 确保使用正确的缩进：
   - 使用2或4个空格进行缩进
   - 避免混合使用空格和制表符
   - 保持同一层级的配置缩进一致

### 3. 更新过时语法

**问题**：`gateway4` 配置已被弃用

**解决方法**：使用新的 `routes` 语法

```yaml
network:
  ethernets:
    enp0s3:  # 网卡名称（根据实际情况修改）
      dhcp4: no
      addresses: [192.168.1.100/24]  # IP地址/子网掩码
      routes:
        - to: default
          via: 192.168.1.1  # 替代gateway4
      nameservers:
        addresses: [8.8.8.8, 114.114.114.114]  # DNS服务器
  version: 2
```

### 4. 启动网络服务

**问题**：网络服务未运行，导致配置无法应用

**解决方法**：
```bash
# 启动并启用systemd-networkd服务
sudo systemctl start systemd-networkd
sudo systemctl enable systemd-networkd

# 重启NetworkManager服务
sudo systemctl restart NetworkManager
```

### 5. 应用配置

**解决方法**：
```bash
sudo netplan apply
```

## 验证网络连接

配置完成后，验证网络是否正常：

```bash
# 查看IP配置
ip addr

# 测试网络连通性
ping -c 4 baidu.com

# 测试DNS解析
nslookup baidu.com

# 查看路由表
ip route
```

## 常见问题排查

1. **网络接口名称确认**：
   ```bash
   ip link
   ```

2. **网关可达性测试**：
   ```bash
   ping 192.168.1.1
   ```

3. **网络服务状态检查**：
   ```bash
   sudo systemctl status systemd-networkd
   sudo systemctl status NetworkManager
   ```

4. **Netplan配置验证**：
   ```bash
   sudo netplan try
   ```

## 技术说明

- **Netplan**：Ubuntu 18.04+ 系统默认的网络配置工具，使用YAML格式配置文件
- **systemd-networkd**：系统守护进程，负责网络配置的应用
- **NetworkManager**：提供更友好的网络管理界面，与systemd-networkd配合使用
- **YAML格式**：对缩进和格式要求严格，是Netplan配置的核心

## 总结

通过以上步骤，可以成功解决Netplan网络配置中的常见问题，确保系统网络正常运行。这些技能对于DevOps工程师来说是基础且必要的，特别是在服务器部署和网络故障排查时。

在遇到网络配置问题时，建议按照以下流程进行排查：
1. 检查配置文件权限和语法
2. 确保使用正确的配置语法
3. 验证网络服务状态
4. 测试网络连通性

通过系统化的排查和解决，可以快速定位并解决网络配置问题。
