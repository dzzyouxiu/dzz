## 一、环境搭建

### 1. 虚拟机软件安装

#### 1.1 VirtualBox安装
- **下载地址**：[https://www.virtualbox.org/wiki/Downloads](https://www.virtualbox.org/wiki/Downloads)

### 3. Shell脚本练习

#### 3.1 基础脚本
- **创建并执行第一个脚本**：
  ```bash
  # 创建hello.sh脚本文件，使用here文档语法
  # 作用：通过cat命令和重定向操作符>创建一个新文件
  # 这里文档(<< 'EOF')允许我们在命令行中直接输入多行内容到文件
  cat > hello.sh << 'EOF'
  #!/bin/bash
  # shebang行：指定脚本使用bash解释器执行
  # 在Linux系统中，这行告诉系统如何执行该脚本文件
  
  # echo命令：在终端输出文本信息
  # 作用：向屏幕显示问候信息
  echo "Hello, DevOps!"
  
  # date命令：显示当前系统的日期和时间
  # 作用：获取并显示系统当前的日期时间
  echo "Current date: $(date)"
  
  # pwd命令：print working directory，显示当前工作目录的绝对路径
  # 作用：告诉用户当前在文件系统中的位置
  echo "Current directory: $(pwd)"
  EOF
  
  # chmod命令：change mode，修改文件权限
  # +x参数：添加执行权限，使脚本可以直接运行
  # 作用：让脚本文件成为可执行文件
  chmod +x hello.sh
  
  # 执行脚本：通过./前缀运行当前目录下的脚本
  # 作用：运行我们创建的hello.sh脚本，执行其中的命令
  ./hello.sh
  ```

#### 3.2 系统监控脚本
- **创建系统监控脚本**：
  ```bash
  # 创建system_monitor.sh脚本文件，使用here文档语法
  # 作用：创建一个用于监控系统状态的脚本文件
  cat > system_monitor.sh << 'EOF'
  #!/bin/bash
  # shebang行：指定脚本使用bash解释器执行
  
  # 输出监控标题
  echo "=== System Monitor ==="
  
  # date命令：显示当前系统日期和时间
  # 作用：记录监控执行的时间点
  echo "Date: $(date)"
  
  # hostname命令：显示系统主机名
  # 作用：标识是哪台服务器的监控信息
  echo "Hostname: $(hostname)"
  echo ""
  
  # 输出CPU使用情况标题
  echo "=== CPU Usage ==="
  
  # top命令：实时显示系统进程资源使用情况
  # -b参数：批处理模式，非交互式运行
  # -n1参数：只执行一次
  # grep "Cpu(s)"：过滤出CPU使用情况的行
  # 作用：查看CPU的整体使用情况
  top -bn1 | grep "Cpu(s)"
  echo ""
  
  # 输出内存使用情况标题
  echo "=== Memory Usage ==="
  
  # free命令：显示内存使用情况
  # -h参数：以人类可读的格式显示（如GB、MB）
  # 作用：查看系统内存的使用情况
  free -h
  echo ""
  
  # 输出磁盘使用情况标题
  echo "=== Disk Usage ==="
  
  # df命令：显示文件系统磁盘空间使用情况
  # -h参数：以人类可读的格式显示
  # 作用：查看各分区的磁盘使用情况
  df -h
  echo ""
  
  # 输出网络连接情况标题
  echo "=== Network Connections ==="
  
  # netstat命令：显示网络连接、路由表等网络相关信息
  # -t参数：显示TCP连接
  # -u参数：显示UDP连接
  # -l参数：显示监听状态的连接
  # -n参数：以数字形式显示地址和端口
  # head -10：只显示前10行输出
  # 作用：查看系统当前的网络连接状态
  netstat -tuln | head -10
  echo ""
  
  # 输出系统负载情况标题
  echo "=== Load Average ==="
  
  # uptime命令：显示系统运行时间和平均负载
  # 作用：查看系统的运行时间和CPU负载情况
  uptime
  EOF
  
  # chmod命令：添加执行权限
  # 作用：使脚本文件可执行
  chmod +x system_monitor.sh
  
  # 执行系统监控脚本
  # 作用：运行监控脚本，显示系统各方面的状态信息
  ./system_monitor.sh
  ```

#### 3.3 备份脚本
- **创建文件备份脚本**：
  ```bash
  # 创建backup.sh脚本文件，使用here文档语法
  # 作用：创建一个用于备份文件的脚本
  cat > backup.sh << 'EOF'
  #!/bin/bash
  # shebang行：指定脚本使用bash解释器执行
  
  # 定义备份源目录和目标目录
  # SOURCE_DIR：要备份的源目录
  # BACKUP_DIR：备份文件存储的目标目录
  SOURCE_DIR="/home"
  BACKUP_DIR="/backup"
  
  # mkdir命令：创建目录
  # -p参数：递归创建目录，即使父目录不存在也不会报错
  # 作用：确保备份目录存在
  mkdir -p $BACKUP_DIR
  
  # date命令：格式化输出当前日期时间
  # +%Y%m%d_%H%M%S：指定日期时间格式为年月日_时分秒
  # 作用：生成唯一的备份文件名，避免覆盖
  DATE=$(date +%Y%m%d_%H%M%S)
  BACKUP_FILE="$BACKUP_DIR/backup_$DATE.tar.gz"
  
  # 输出备份开始信息
  echo "Starting backup from $SOURCE_DIR to $BACKUP_FILE"
  
  # tar命令：打包和压缩文件
  # -c参数：创建新的归档文件
  # -z参数：使用gzip压缩
  # -f参数：指定归档文件名
  # 作用：将源目录打包压缩成一个tar.gz文件
  tar -czf $BACKUP_FILE $SOURCE_DIR
  
  # 检查备份是否成功
  # $?：上一个命令的退出状态码，0表示成功
  # if-else语句：条件判断
  # 作用：验证备份操作是否成功完成
  if [ $? -eq 0 ]; then
      echo "Backup completed successfully!"
      # du命令：显示文件或目录的磁盘使用情况
      # -h参数：以人类可读的格式显示
      # 作用：显示备份文件的大小
      echo "Backup size: $(du -h $BACKUP_FILE)"
  else
      echo "Backup failed!"
      # exit命令：退出脚本
      # 1：表示错误退出状态码
      # 作用：备份失败时以错误状态退出
      exit 1
  fi
  
  # 保留最近5个备份，删除旧备份
  # cd命令：切换到备份目录
  cd $BACKUP_DIR
  
  # ls命令：列出文件
  # -t参数：按修改时间排序，最新的在前
  # backup_*.tar.gz：匹配所有以backup_开头的tar.gz文件
  # tail -n +6：从第6行开始显示，即跳过前5个文件
  # xargs rm -f：将前面命令的输出作为参数传递给rm命令，强制删除
  # 作用：只保留最近5个备份文件，删除更早的备份
  ls -t backup_*.tar.gz | tail -n +6 | xargs rm -f
  echo "Old backups cleaned up."
  EOF
  
  # chmod命令：添加执行权限
  # 作用：使脚本文件可执行
  chmod +x backup.sh
  
  # 执行备份脚本
  # 作用：运行备份脚本，执行文件备份操作
  ./backup.sh
  ```

### 项目一：基础服务搭建

**目标**：搭建一个包含HTTP、SSH服务的基础Linux服务器

**原理**：
- HTTP服务：提供网页访问功能，使用Apache或Nginx作为Web服务器
- SSH服务：提供远程登录功能，使用OpenSSH实现
- 防火墙：保护服务器安全，只开放必要的端口

**步骤**：

#### 1. 安装并配置HTTP服务（Apache）

**CentOS系统**：
```bash
# 安装Apache服务
yum install -y httpd

# 启动Apache服务
systemctl start httpd

# 设置Apache服务开机自启
systemctl enable httpd

# 查看Apache服务状态
systemctl status httpd
```

**Ubuntu系统**：
```bash
# 安装Apache服务
sudo apt update
sudo apt install -y apache2

# 启动Apache服务
sudo systemctl start apache2

# 设置Apache服务开机自启
sudo systemctl enable apache2

# 查看Apache服务状态
sudo systemctl status apache2
```

#### 2. 安装并配置SSH服务

**CentOS系统**：
```bash
# 安装OpenSSH服务
# 通常CentOS默认已安装

# 启动SSH服务
systemctl start sshd

# 设置SSH服务开机自启
systemctl enable sshd

# 查看SSH服务状态
systemctl status sshd
```

**Ubuntu系统**：
```bash
# 安装OpenSSH服务
sudo apt update
sudo apt install -y openssh-server

# 启动SSH服务
sudo systemctl start ssh

# 设置SSH服务开机自启
sudo systemctl enable ssh

# 查看SSH服务状态
sudo systemctl status ssh
```

#### 3. 配置防火墙，开放相关端口

**CentOS系统（使用firewalld）**：

### 项目二：系统监控平台

**目标**：搭建一个简单的系统监控平台

**原理**：
- 系统监控脚本：使用shell脚本收集系统各项指标
- 定时任务：使用crontab定期执行监控脚本
- 日志记录：将监控结果输出到日志文件，便于分析
- 性能分析：通过查看监控数据，识别系统性能瓶颈

**步骤**：

#### 1. 创建系统监控脚本

```bash
# 创建系统监控脚本文件
cat > system_monitor.sh << 'EOF'
#!/bin/bash
# 系统监控脚本

# 定义日志文件路径
LOG_FILE="/var/log/system_monitor.log"

# 创建日志目录（如果不存在）
mkdir -p $(dirname $LOG_FILE)

# 输出监控时间
echo "=== System Monitor - $(date) ===" >> $LOG_FILE

# 输出系统信息
echo "Hostname: $(hostname)" >> $LOG_FILE
echo "Kernel version: $(uname -r)" >> $LOG_FILE
echo "" >> $LOG_FILE

# 监控CPU使用情况
echo "=== CPU Usage ===" >> $LOG_FILE
top -bn1 | grep "Cpu(s)" >> $LOG_FILE
echo "" >> $LOG_FILE

# 监控内存使用情况
echo "=== Memory Usage ===" >> $LOG_FILE
free -h >> $LOG_FILE
echo "" >> $LOG_FILE

# 监控磁盘使用情况
echo "=== Disk Usage ===" >> $LOG_FILE
df -h >> $LOG_FILE
echo "" >> $LOG_FILE

# 监控网络连接
echo "=== Network Connections ===" >> $LOG_FILE
netstat -tuln | head -10 >> $LOG_FILE
echo "" >> $LOG_FILE

# 监控系统负载
echo "=== System Load ===" >> $LOG_FILE
uptime >> $LOG_FILE
echo "" >> $LOG_FILE

# 监控进程情况
echo "=== Top 5 Processes by CPU Usage ===" >> $LOG_FILE
ps aux --sort=-%cpu | head -6 >> $LOG_FILE
echo "" >> $LOG_FILE

echo "=== Top 5 Processes by Memory Usage ===" >> $LOG_FILE
ps aux --sort=-%mem | head -6 >> $LOG_FILE
echo "" >> $LOG_FILE

# 输出监控结束标记
echo "=== Monitoring Completed ===" >> $LOG_FILE
echo "" >> $LOG_FILE
echo "" >> $LOG_FILE
EOF

# 添加执行权限
chmod +x system_monitor.sh

# 测试运行监控脚本
./system_monitor.sh

# 查看日志输出
tail -f /var/log/system_monitor.log
```

#### 2. 配置定时任务，定期执行监控

```bash
# 编辑crontab文件
crontab -e

# 在打开的编辑器中添加以下内容（每15分钟执行一次监控）
# 每15分钟执行一次系统监控
*/15 * * * * /path/to/system_monitor.sh

# 保存并退出编辑器
# 在vi编辑器中，按Esc键，然后输入:wq保存退出
```

#### 3. 查看监控日志，分析系统状态

```bash
# 实时查看监控日志的最新内容
tail -f /var/log/system_monitor.log
```

### 项目三：文件备份系统

**目标**：搭建一个自动文件备份系统

**原理**：
- 备份脚本：使用shell脚本和tar命令实现文件打包压缩
- 定时任务：使用crontab定期执行备份脚本
- 备份策略：保留最近的备份文件，自动清理旧备份
- 恢复功能：使用tar命令从备份文件中恢复数据

**步骤**：

#### 1. 创建备份脚本

```bash
# 创建备份脚本文件
cat > backup.sh << 'EOF'
#!/bin/bash
# 文件备份脚本

# 定义备份源目录
SOURCE_DIR="/home"

# 定义备份目标目录
BACKUP_DIR="/backup"

# 定义保留的备份文件数量
KEEP_BACKUPS=5

# 创建备份目录（如果不存在）
mkdir -p $BACKUP_DIR

# 生成备份文件名（包含日期时间戳）
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_FILE="$BACKUP_DIR/backup_$DATE.tar.gz"

# 输出备份开始信息
echo "Starting backup from $SOURCE_DIR to $BACKUP_FILE"

# 执行备份（使用tar命令打包压缩）
tar -czf $BACKUP_FILE $SOURCE_DIR

# 检查备份是否成功
if [ $? -eq 0 ]; then
    echo "Backup completed successfully!"
    # 显示备份文件大小
    echo "Backup size: $(du -h $BACKUP_FILE)"
else
    echo "Backup failed!"
    exit 1
fi

# 清理旧备份，只保留最近的KEEP_BACKUPS个备份
 echo "Cleaning up old backups..."
cd $BACKUP_DIR

# 按修改时间排序，保留最近的KEEP_BACKUPS个备份，删除其他备份
ls -t backup_*.tar.gz | tail -n +$((KEEP_BACKUPS + 1)) | xargs -r rm -f

# 显示清理结果
echo "Old backups cleaned up."
echo "Current backups:"
ls -l backup_*.tar.gz
EOF

# 添加执行权限
chmod +x backup.sh

# 测试运行备份脚本
./backup.sh

# 查看备份文件
ls -l /backup/
```

#### 2. 配置定时任务，定期执行备份

```bash
# 编辑crontab文件
crontab -e

# 在打开的编辑器中添加以下内容（每天凌晨2点执行备份）
# 每天凌晨2点执行备份
0 2 * * * /path/to/backup.sh >> /var/log/backup.log 2>&1

# 保存并退出编辑器
# 在vi编辑器中，按Esc键，然后输入:wq保存退出
```

#### 3. 验证备份文件是否生成

```bash
# 查看备份目录中的文件
ls -l /backup/

# 查看备份日志
cat /var/log/backup.log

# 查看备份文件大小
du -h /backup/*.tar.gz
```

#### 4. 测试备份恢复功能

**模拟数据丢失**：
```bash
# 创建测试文件
echo "Test file" > /home/test_file.txt

# 执行备份
./backup.sh

# 删除测试文件
rm /home/test_file.txt

# 确认文件已删除
ls -l /home/test_file.txt
```

**从备份恢复**：
```bash
# 查看最新的备份文件
LATEST_BACKUP=$(ls -t /backup/backup_*.tar.gz | head -1)
echo "Restoring from backup: $LATEST_BACKUP"

# 创建临时恢复目录
mkdir -p /tmp/restore

# 从备份文件中恢复到临时目录
tar -xzf $LATEST_BACKUP -C /tmp/restore

# 查看恢复的文件
ls -l /tmp/restore/home/

# 将恢复的文件复制回原位置
cp /tmp/restore/home/test_file.txt /home/

# 确认文件已恢复
ls -l /home/test_file.txt
cat /home/test_file.txt
```

**验证**：
- 查看备份文件：`ls -l /backup/` - 应显示生成的备份文件
- 测试恢复：`tar -xzf backup_file.tar.gz -C /tmp/` - 应能成功恢复文件
- 定时任务执行：第二天查看备份目录，应有新的备份文件
- 备份清理：备份目录中应只保留最近5个备份文件

## 四、故障排除练习

### 1. 网络故障排查

**场景**：无法访问外部网络

**排查步骤**：
1. 检查网络接口状态：`ifconfig`
2. 检查IP地址配置：`ip addr`
3. 检查默认网关：`ip route`
4. 测试网关连通性：`ping 网关IP`
5. 测试DNS服务器：`ping 8.8.8.8`
6. 检查DNS配置：`cat /etc/resolv.conf`

### 2. 服务故障排查

**场景**：HTTP服务无法访问

**排查步骤**：
1. 检查服务状态：`systemctl status httpd`
2. 检查服务端口：`netstat -tuln | grep 80`
3. 检查防火墙规则：`firewall-cmd --list-ports`
4. 检查服务日志：`tail -f /var/log/httpd/error_log`
5. 测试本地访问：`curl http://localhost`

### 3. 磁盘空间故障排查

**场景**：磁盘空间不足

**排查步骤**：
1. 检查磁盘使用情况：`df -h`
2. 查找大文件：`find / -type f -size +100M | sort -nr`
3. 查找大目录：`du -h --max-depth=1 /`
4. 清理临时文件：`rm -rf /tmp/*`
5. 清理日志文件：`cat /dev/null > /var/log/old.log`
