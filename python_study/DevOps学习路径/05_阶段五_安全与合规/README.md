# 阶段五：安全与合规

## 学习目标
- 理解DevSecOps理念
- 掌握常见安全工具和技术
- 能够在CI/CD中集成安全扫描
- 了解合规和审计要求

## 主要内容
1. **安全基础**：网络安全、身份认证、权限管理
2. **代码安全**：代码扫描、依赖检查、密钥管理
3. **容器安全**：镜像扫描、运行时安全
4. **合规审计**：日志审计、合规检查、安全策略

## 学习路径

### 第1-2周：安全基础

#### 知识点
- CIA三元组（机密性、完整性、可用性）
- 身份认证和访问控制
- 网络安全基础（防火墙、WAF）
- 数据加密（传输加密、存储加密）
- 安全最佳实践

#### 实操练习
1. 配置SSH安全
2. 配置防火墙规则
3. 实施密码策略
4. 配置HTTPS证书

#### 免费资源
- OWASP Top 10：https://owasp.org/www-project-top-ten
- NIST安全框架：https://www.nist.gov/cyberframework

### 第3-4周：代码安全

#### 知识点
- SAST（静态应用安全测试）
- DAST（动态应用安全测试）
- 依赖漏洞扫描
- 密钥和敏感信息管理
- Git安全（敏感信息泄露）

#### 工具学习
- SonarQube（代码质量和安全）
- Snyk（依赖扫描）
- Trivy（漏洞扫描）
- GitGuardian（密钥检测）

#### 实操练习
1. 使用SonarQube扫描代码
2. 检查npm/pip依赖漏洞
3. 清理Git历史中的敏感信息
4. 配置敏感信息管理

### 第5-6周：容器安全

#### 知识点
- 容器镜像安全
- 容器运行时安全
- Kubernetes安全
- 镜像签名和验证

#### 工具学习
- Trivy（镜像扫描）
- Clair（镜像漏洞）
- Falco（运行时安全）
- OPA（策略引擎）

#### 实操练习
1. 扫描Docker镜像漏洞
2. 配置镜像仓库安全
3. 加固容器运行时
4. 配置Kubernetes安全策略

### 第7-8周：合规审计

#### 知识点
- 常见合规标准（ISO27001、SOC2、GDPR）
- 审计日志管理
- 安全策略执行
- 安全事件响应

#### 实操练习
1. 配置集中日志审计
2. 实现变更审计
3. 编写安全策略文档
4. 模拟安全事件响应

## 学习方法
1. **左移安全**：在开发早期集成安全
2. **工具自动化**：使用工具自动检查
3. **持续学习**：跟踪新的安全威胁
4. **实战演练**：模拟攻击和响应

## 时间规划
- 总时长：8周
- 每天学习时间：2-3小时
- 周末完成综合实验

## 预期成果
- 理解DevSecOps理念和实践
- 能够使用安全工具进行扫描
- 能够在CI/CD中集成安全检查
- 了解基本的合规要求

## 后续阶段
完成本阶段后，将进入**阶段六：高级技能**，学习混沌工程、性能优化等高级主题。
