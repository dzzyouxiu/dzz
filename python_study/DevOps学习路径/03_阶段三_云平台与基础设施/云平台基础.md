# 云平台基础

## 云计算概述

### 服务模型

#### IaaS（基础设施即服务）

- 提供虚拟化的计算资源
- 用户管理：操作系统、存储、网络
- 服务商管理：服务器、存储、网络硬件
- 代表服务：AWS EC2、阿里云ECS、腾讯云CVM

#### PaaS（平台即服务）

- 提供开发和部署平台
- 用户管理：应用程序和数据
- 服务商管理：操作系统、运行时、中间件
- 代表服务：AWS Elastic Beanstalk、阿里云SAE

#### SaaS（软件即服务）

- 提供完整的软件应用
- 用户只需要使用软件
- 服务商管理：所有基础设施和应用
- 代表服务：Office 365、Salesforce

### 部署模型

| 模型  | 描述       | 适用场景       |
| --- | -------- | ---------- |
| 公有云 | 第三方提供商提供 | 初创公司、测试环境  |
| 私有云 | 企业自建自用   | 高安全要求、合规需求 |
| 混合云 | 公有云+私有云  | 数据敏感、弹性扩展  |
| 多云  | 多个公有云提供商 | 避免厂商锁定、容灾  |

## 计算服务

### Ubuntu操作系统介绍

Ubuntu是一款基于Debian的开源Linux操作系统，广泛应用于云平台和服务器环境。

#### 主要特点
- 开源免费
- 长期支持版本（LTS）提供5年技术支持
- 丰富的软件包生态系统
- 强大的社区支持
- 良好的安全性和稳定性
- 适合从个人电脑到企业服务器的各种场景

#### Ubuntu LTS版本
- Ubuntu 20.04 LTS（Focal Fossa）- 2020年4月发布，支持至2025年4月
- Ubuntu 22.04 LTS（Jammy Jellyfish）- 2022年4月发布，支持至2027年4月
- Ubuntu 24.04 LTS（Noble Numbat）- 2024年4月发布，支持至2029年4月

### 实例类型

#### 通用型（General Purpose）

- 平衡的计算、内存和网络资源
- 适用于Web服务器、应用服务器
- AWS：t系列、m系列
- 阿里云：g系列、e系列

#### 计算优化型（Compute Optimized）

- 高CPU性能
- 适用于批处理、科学计算
- AWS：c系列
- 阿里云：c系列

#### 内存优化型（Memory Optimized）

- 大内存配置
- 适用于数据库、内存缓存
- AWS：r系列、x系列
- 阿里云：r系列、re系列

#### 存储优化型（Storage Optimized）

- 高存储吞吐量
- 适用于大数据、数据仓库
- AWS：i系列、d系列
- 阿里云：i系列、d系列

## 存储服务

### 对象存储（Object Storage）

#### 特点

- 无限扩展
- 按使用付费
- HTTP/HTTPS访问
- 高持久性（99.999999999%）

#### 使用场景

- 静态网站托管
- 备份和恢复
- 大数据分析
- 内容分发

#### 常用操作

```bash
# AWS S3操作
aws s3 mb s3://my-bucket-name           # 创建桶
aws s3 cp local-file.txt s3://my-bucket/  # 上传文件
aws s3 ls s3://my-bucket                  # 列出文件
aws s3 sync ./local-dir s3://my-bucket    # 同步目录

# 阿里云OSS操作
ossutil mb oss://my-bucket --endpoint oss-cn-hangzhou.aliyuncs.com
ossutil cp local-file.txt oss://my-bucket/
ossutil ls oss://my-bucket
```

### 块存储（Block Storage）

#### 特点

- 挂载到实例
- 像本地硬盘一样使用
- 支持快照
- 支持扩展

#### 类型

| 类型    | 性能     | 适用场景   |
| ----- | ------ | ------ |
| 普通云盘  | 低IOPS  | 冷数据、备份 |
| 高效云盘  | 中等IOPS | 开发测试   |
| SSD云盘 | 高IOPS  | 数据库、生产 |
| 本地SSD | 极高IOPS | 高性能数据库 |

### 文件存储（File Storage）

#### 特点

- NFS/SMB协议
- 多实例共享
- 完全托管

#### 适用场景

- 共享存储
- 内容管理
- 开发环境

## 网络服务

### VPC（虚拟私有云）

#### 核心组件

```
VPC (10.0.0.0/16)
├── 公网子网A (10.0.1.0/24) - 可用区A
│   ├── Internet网关
│   └── 公网路由表
├── 公网子网B (10.0.2.0/24) - 可用区B
├── 私网子网A (10.0.10.0/24) - 可用区A
│   ├── NAT网关
│   └── 私网路由表
└── 私网子网B (10.0.20.0/24) - 可用区B
```

#### CIDR规划

```
推荐的VPC划分：
- VPC：10.0.0.0/16（65536个IP）
- 公网子网：10.0.1.0/24、10.0.2.0/24
- 私网子网：10.0.10.0/24、10.0.20.0/24
- 预留：10.0.100.0/24（未来扩展）
```

### 安全组（Security Group）

#### 特点

- 有状态防火墙
- 实例级别防护
- 白名单机制
- 可动态更新

#### 示例配置

| 方向 | 协议  | 端口  | 源IP       | 说明          |
| -- | --- | --- | --------- | ----------- |
| 入站 | TCP | 22  | 0.0.0.0/0 | SSH（建议限制IP） |
| 入站 | TCP | 80  | 0.0.0.0/0 | HTTP        |
| 入站 | TCP | 443 | 0.0.0.0/0 | HTTPS       |
| 出站 | ALL | ALL | 0.0.0.0/0 | 允许所有出站      |

## 实操练习

### 练习1：搭建基础Web环境

**步骤1：创建VPC和子网**
**使用阿里云控制台**：
1. 登录阿里云控制台 → 云服务器ECS → 网络与安全 → 专有网络VPC
2. 点击"创建专有网络"
   - 名称：web-vpc
   - 地域：选择靠近您的区域
   - IPv4网段：10.0.0.0/16
   - 点击"确定"
3. 在VPC详情页点击"创建子网"
   - 名称：web-subnet
   - 可用区：选择一个可用区
   - 子网IPv4网段：10.0.1.0/24
   - 点击"确定"

**使用阿里云CLI**：
```bash
# 创建VPC
aliyun ecs CreateVpc --RegionId cn-hangzhou --CidrBlock 10.0.0.0/16 --VpcName web-vpc

# 创建子网
aliyun ecs CreateVSwitch --RegionId cn-hangzhou --VpcId vpc-xxx --ZoneId cn-hangzhou-a --CidrBlock 10.0.1.0/24 --VSwitchName web-subnet
```

**步骤2：创建安全组**
**使用阿里云控制台**：
1. 云服务器ECS → 网络与安全 → 安全组
2. 点击"创建安全组"
   - 模板：自定义
   - 安全组名称：web-sg
   - 网络类型：专有网络
   - 专有网络：选择刚才创建的web-vpc
   - 点击"确定"
3. 点击安全组名称进入详情页
4. 点击"入方向" → "快速添加"
   - 勾选"HTTP(80)"、"HTTPS(443)"、"SSH(22)"
   - 源地址：0.0.0.0/0（生产环境建议限制IP）
   - 点击"确定"

**使用阿里云CLI**：
```bash
# 创建安全组
aliyun ecs CreateSecurityGroup --RegionId cn-hangzhou --SecurityGroupName web-sg --Description "Web服务器安全组" --VpcId vpc-xxx

# 添加安全组规则
aliyun ecs AuthorizeSecurityGroup --RegionId cn-hangzhou --SecurityGroupId sg-xxx --IpProtocol tcp --PortRange 80/80 --SourceCidrIp 0.0.0.0/0
aliyun ecs AuthorizeSecurityGroup --RegionId cn-hangzhou --SecurityGroupId sg-xxx --IpProtocol tcp --PortRange 443/443 --SourceCidrIp 0.0.0.0/0
aliyun ecs AuthorizeSecurityGroup --RegionId cn-hangzhou --SecurityGroupId sg-xxx --IpProtocol tcp --PortRange 22/22 --SourceCidrIp 0.0.0.0/0
```

**步骤3：创建ECS实例**
**使用阿里云控制台**：
1. 云服务器ECS → 实例 → 创建实例
2. 配置：
   - 计费方式：按量付费
   - 地域：与VPC同一地域
   - 实例规格：ecs.g6.large（2核4G）
   - 镜像：Ubuntu 20.04
   - 网络：选择刚才创建的VPC和子网
   - 公网IP：分配公网IPv4地址
   - 安全组：选择刚才创建的web-sg
   - 登录凭证：设置密码或使用密钥对
   - 实例名称：web-server
3. 点击"下一步" → "创建实例"

**使用阿里云CLI**：
```bash
# 创建ECS实例
aliyun ecs RunInstances \
  --RegionId cn-hangzhou \
  --ImageId ubuntu_20_04_x64_20G_alibase_20230518.vhd \
  --InstanceType ecs.g6.large \
  --SecurityGroupId sg-xxx \
  --VSwitchId vsw-xxx \
  --InternetMaxBandwidthOut 10 \
  --InstanceName web-server \
  --Password YourPassword123!
```

**步骤4：安装Nginx**
1. 使用SSH连接到ECS实例
   ```bash
   ssh root@公网IP
   ```
2. 安装Nginx
   ```bash
   # 更新系统
   apt update -y
   
   # 安装Nginx
   apt install -y nginx
   
   # 启动Nginx
   systemctl start nginx
   systemctl enable nginx
   ```

**步骤5：访问验证**
- 在浏览器中输入ECS实例的公网IP
- 验证是否看到Nginx欢迎页面

### 练习2：使用对象存储托管静态网站

**步骤1：创建存储桶**
**使用阿里云控制台**：
1. 登录阿里云控制台 → 对象存储OSS
2. 点击"创建存储空间"
   - 存储空间名称：your-static-website（全局唯一）
   - 地域：选择与ECS同一地域
   - 存储类型：标准存储
   - 读写权限：公共读
   - 点击"确定"

**使用OSS CLI**：
```bash
# 安装ossutil
curl -o ossutil64 https://gosspublic.alicdn.com/ossutil/1.7.16/ossutil64
chmod +x ossutil64

# 配置ossutil
./ossutil64 config

# 创建存储桶
./ossutil64 mb oss://your-static-website --endpoint oss-cn-hangzhou.aliyuncs.com
```

**步骤2：启用静态网站托管**
**使用阿里云控制台**：
1. 进入存储桶详情页 → 基础设置 → 静态网站托管
2. 点击"设置"
   - 状态：开启
   - 索引文档：index.html
   - 错误文档：404.html
   - 点击"保存"

**使用OSS CLI**：
```bash
# 启用静态网站托管
./ossutil64 website oss://your-static-website --index index.html --error 404.html
```

**步骤3：创建并上传index.html**
1. 创建index.html文件
   ```html
   <!DOCTYPE html>
   <html>
   <head>
       <title>静态网站</title>
       <style>
           body {
               font-family: Arial, sans-serif;
               text-align: center;
               padding: 50px;
           }
       </style>
   </head>
   <body>
       <h1>欢迎访问我的静态网站</h1>
       <p>这是通过阿里云OSS托管的静态网站</p>
   </body>
   </html>
   ```

2. 上传文件
   ```bash
   # 上传index.html
   ./ossutil64 cp index.html oss://your-static-website/
   
   # 上传404.html（可选）
   echo "<h1>404 Not Found</h1>" > 404.html
   ./ossutil64 cp 404.html oss://your-static-website/
   ```

**步骤4：配置公开访问权限**
**使用阿里云控制台**：
1. 存储桶详情页 → 权限管理 → 访问控制
2. 确保读写权限设置为"公共读"

**使用OSS CLI**：
```bash
# 设置存储桶为公共读
./ossutil64 set-acl oss://your-static-website public-read
```

**步骤5：访问验证**
- 在存储桶详情页 → 基础设置 → 静态网站托管
- 复制"访问域名"
- 在浏览器中访问该域名，验证是否看到自定义页面

### 练习3：数据库迁移上云

**步骤1：创建RDS实例**
**使用阿里云控制台**：
1. 登录阿里云控制台 → 云数据库RDS
2. 点击"创建实例"
3. 配置：
   - 计费方式：按量付费
   - 地域：与ECS同一地域
   - 数据库类型：MySQL
   - 版本：8.0
   - 规格：2核4G
   - 存储空间：20GB
   - 网络类型：专有网络
   - VPC：选择与ECS同一VPC
   - 安全组：创建新安全组或选择现有安全组
   - 数据库账号：
     - 账号名：root
     - 密码：设置强密码
   - 点击"创建实例"

**使用阿里云CLI**：
```bash
# 创建RDS实例
aliyun rds CreateDBInstance \
  --RegionId cn-hangzhou \
  --Engine MySQL \
  --EngineVersion 8.0 \
  --DBInstanceClass mysql.n2.small.1 \
  --DBInstanceStorage 20 \
  --VPCId vpc-xxx \
  --VSwitchId vsw-xxx \
  --SecurityIPList 10.0.1.0/24 \
  --MasterUserPassword YourPassword123!
```

**步骤2：配置安全组**
**使用阿里云控制台**：
1. RDS实例详情页 → 安全组 → 配置安全组
2. 确保安全组允许ECS实例的IP访问3306端口

**使用阿里云CLI**：
```bash
# 为RDS安全组添加规则
aliyun ecs AuthorizeSecurityGroup \
  --RegionId cn-hangzhou \
  --SecurityGroupId sg-xxx \
  --IpProtocol tcp \
  --PortRange 3306/3306 \
  --SourceCidrIp 10.0.1.0/24
```

**步骤3：从本地数据库导出数据**
1. 在本地服务器执行：
   ```bash
   # 导出数据库
   mysqldump -u root -p --databases your_database > your_database.sql
   ```

2. 将导出文件上传到ECS实例：
   ```bash
   scp your_database.sql root@ECS公网IP:/tmp/
   ```

**步骤4：导入到RDS**
1. SSH连接到ECS实例
2. 安装MySQL客户端：
   ```bash
   apt install -y mysql-client
   ```

3. 导入数据到RDS：
   ```bash
   # 连接RDS并创建数据库
   mysql -h RDS内网地址 -u root -p -e "CREATE DATABASE IF NOT EXISTS your_database;"
   
   # 导入数据
   mysql -h RDS内网地址 -u root -p your_database < /tmp/your_database.sql
   ```

**步骤5：修改应用连接配置**
1. 编辑应用的数据库配置文件，将连接地址改为RDS的内网地址
2. 更新数据库用户名和密码
3. 重启应用服务

**步骤6：验证**
- 访问应用，测试数据库连接
- 检查数据是否完整
- 执行一些数据库操作，确保一切正常

### 练习4：在Ubuntu上安装Docker和Docker Compose

**步骤1：更新系统**
```bash
# 更新包索引
apt update -y

# 升级已安装的包
apt upgrade -y
```

**步骤2：安装依赖包**
```bash
# 安装必要的依赖包
apt install -y apt-transport-https ca-certificates curl gnupg lsb-release
```

**步骤3：添加Docker官方GPG密钥**
```bash
# 添加Docker GPG密钥
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg
```

**步骤4：添加Docker仓库**
```bash
# 添加Docker稳定版仓库
echo "deb [arch=amd64 signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | tee /etc/apt/sources.list.d/docker.list > /dev/null
```

**步骤5：安装Docker**

Docker有两种安装方式：

**方式1：从Ubuntu官方仓库安装（简单）**
```bash
# 直接安装docker.io包
apt install -y docker.io

# 启动Docker服务
systemctl start docker
systemctl enable docker
```

**方式2：从Docker官方仓库安装（推荐）**
```bash
# 更新包索引
apt update -y

# 安装Docker引擎
apt install -y docker-ce docker-ce-cli containerd.io

# 启动Docker服务
systemctl start docker
systemctl enable docker
```

**两种安装方式的区别**：
- **包来源**：docker.io来自Ubuntu官方仓库，docker-ce来自Docker官方仓库
- **版本**：docker.io通常是较旧的版本，docker-ce提供最新的稳定版本
- **功能**：docker-ce包含更多最新功能和安全更新
- **支持**：docker-ce有Docker官方的直接支持
- **安装复杂度**：docker.io安装更简单，docker-ce安装步骤更多但更可靠

**推荐使用方式2**，因为它提供更稳定、更安全的Docker版本，尤其是在生产环境中。

**步骤6：验证Docker安装**
```bash
# 检查Docker版本
docker --version

# 运行Hello World容器验证安装
docker run hello-world
```

**步骤7：安装Docker Compose**
```bash
# 下载Docker Compose（版本号可根据需要调整）
curl -L "https://github.com/docker/compose/releases/download/1.29.2/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose

# 赋予执行权限
chmod +x /usr/local/bin/docker-compose

# 创建软链接
ln -s /usr/local/bin/docker-compose /usr/bin/docker-compose
```

**步骤8：验证Docker Compose安装**
```bash
# 检查Docker Compose版本
docker-compose --version
```

**步骤9：配置Docker用户权限（可选）**
```bash
# 将当前用户添加到docker组
groupadd docker
usermod -aG docker $USER

# 重新登录以应用权限变化
```

### 练习5：在Ubuntu上安装Kali Linux工具包

Kali Linux是一个专为网络安全测试和渗透测试设计的Linux发行版。在Ubuntu上，我们可以安装Kali Linux的工具包，而不需要完整安装Kali Linux。

**步骤1：添加Kali Linux仓库**
```bash
# 编辑sources.list文件
nano /etc/apt/sources.list

# 在文件末尾添加以下行（适用于Ubuntu 20.04）
deb http://http.kali.org/kali kali-rolling main non-free contrib
```

**步骤2：添加Kali Linux GPG密钥**
```bash
# 下载并添加Kali Linux GPG密钥
wget -q -O - https://archive.kali.org/archive-key.asc | apt-key add -
```

**步骤3：更新包索引**
```bash
# 更新包索引
apt update
```

**步骤4：安装Kali Linux工具包**

根据需要选择安装方式：

**安装特定工具**
```bash
# 安装单个工具，例如nmap
apt install -y nmap

# 安装一组工具，例如信息收集工具
apt install -y kali-tools-information-gathering
```

**安装所有Kali工具（不推荐，会占用大量空间）**
```bash
# 安装完整的Kali工具集
apt install -y kali-linux-full
```

## 成本优化建议

### 计算成本

- 使用按需实例用于开发测试
- 使用预留实例用于稳定负载
- 使用竞价实例用于批处理
- 及时关闭不用的实例

### 存储成本

- 使用生命周期策略自动归档
- 清理无用的快照和备份
- 使用低频存储归档冷数据
- 压缩存储数据

### 网络成本

- 同区域内流量免费
- 使用内网传输减少公网流量
- 合理规划跨区域流量
