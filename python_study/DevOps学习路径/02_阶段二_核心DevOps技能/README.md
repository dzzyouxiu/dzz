# 阶段二：核心DevOps技能

## 学习目标
- 掌握Git版本控制
- 熟练使用CI/CD工具
- 理解并应用Docker容器技术
- 掌握Kubernetes容器编排

## 主要内容
1. **Git版本控制**：仓库管理、分支操作、合并冲突解决
2. **CI/CD实践**：Jenkins、GitLab CI、GitHub Actions
3. **Docker容器**：镜像、容器、Dockerfile、Docker Compose
4. **Kubernetes编排**：Pod、Service、Deployment、Helm

## 学习路径

### 第1-2周：Git版本控制
- Git基础操作
- 分支管理
- 远程仓库
- Git工作流

### 第3-4周：CI/CD实践
- Jenkins安装配置
- Pipeline编写
- GitLab CI配置
- GitHub Actions使用

### 第5-6周：Docker容器
- Docker基础
- Dockerfile编写
- Docker Compose
- 容器网络和存储

### 第7-8周：Kubernetes编排
- Kubernetes基础
- 核心概念
- 部署应用
- Helm使用

## 学习方法
1. **理论学习**：阅读官方文档和相关书籍
2. **实践操作**：搭建实验环境，进行实际操作
3. **项目练习**：通过实际项目巩固所学知识
4. **社区参与**：关注DevOps社区，学习最佳实践

## 时间规划
- 总时长：8周
- 每天学习时间：2-3小时
- 周末可以安排较长时间的实验操作

## 预期成果
- 能够熟练使用Git进行版本控制
- 能够搭建完整的CI/CD流水线
- 能够使用Docker构建和管理容器
- 能够在Kubernetes上部署和管理应用

## 后续阶段
完成本阶段学习后，将进入**阶段三：监控与可观测性**，学习Prometheus、Grafana、ELK Stack等监控工具。
