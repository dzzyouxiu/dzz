# 1. 安全基础概念（2周）

## 学习目标

- 理解网络安全的基本概念和原则
- 掌握常见的攻击类型和防御方法
- 熟悉OWASP Top 10漏洞原理
- 了解常见端口和服务的安全风险

## 学习内容

### 1.1 CIA三要素

- **机密性（Confidentiality）**：确保信息不被未授权访问
  - 加密技术：对称加密、非对称加密
  - 访问控制：身份认证、权限管理
  - 数据保护：传输加密、存储加密
- **完整性（Integrity）**：确保信息不被未授权修改
  - 哈希算法：MD5、SHA-256
  - 数字签名：确保数据来源和完整性
  - 防篡改技术：数据校验、版本控制
- **可用性（Availability）**：确保系统和服务持续可用
  - 冗余设计：多备份、负载均衡
  - 灾难恢复：备份策略、恢复计划
  - 抗拒绝服务：流量过滤、限流

### 1.2 常见攻击类型

#### SQL注入

- **原理**：通过在用户输入中插入SQL语句，操纵数据库查询
- **危害**：获取敏感数据、修改数据库、执行系统命令
- **防御**：参数化查询、输入验证、最小权限

#### 跨站脚本（XSS）

- **原理**：在网页中注入恶意脚本，当用户访问时执行
- **类型**：反射型、存储型、DOM型
- **危害**：窃取cookie、会话劫持、钓鱼
- **防御**：输入验证、输出编码、CSP策略

#### 跨站请求伪造（CSRF）

- **原理**：利用用户已认证的会话，诱导用户执行非预期操作
- **危害**：未授权操作、数据篡改
- **防御**：CSRF Token、SameSite Cookie、验证Referer

#### 文件上传漏洞

- **原理**：上传恶意文件到服务器，执行恶意代码
- **危害**：服务器被控制、webshell上传
- **防御**：文件类型验证、文件内容检查、存储路径隔离

#### 命令注入

- **原理**：在用户输入中插入系统命令，服务器执行
- **危害**：系统被控制、数据泄露
- **防御**：输入验证、命令白名单、最小权限

### 1.3 OWASP Top 10漏洞原理

| 排名 | 漏洞类型          | 描述              | 危害         |
| -- | ------------- | --------------- | ---------- |
| 1  | 失效的访问控制       | 限制对资源的访问失败      | 未授权访问、数据泄露 |
| 2  | 加密机制失效        | 密码和敏感数据保护不足     | 数据泄露、身份盗用  |
| 3  | 注入            | 如SQL、NoSQL、命令注入 | 数据泄露、系统控制  |
| 4  | 不安全设计         | 设计层面的安全缺陷       | 多种安全问题     |
| 5  | 安全配置错误        | 安全设置不当          | 未授权访问、数据泄露 |
| 6  | 易受攻击的组件       | 使用有漏洞的库和框架      | 多种安全问题     |
| 7  | 身份认证与授权失效     | 身份验证机制存在缺陷      | 未授权访问      |
| 8  | 软件与数据完整性失效    | 代码和数据未验证        | 恶意代码执行     |
| 9  | 安全日志与监控失效     | 日志记录和监控不足       | 攻击无法及时发现   |
| 10 | 服务端请求伪造（SSRF） | 服务器被诱导访问内部资源    | 内部系统访问     |

### 1.4 常见端口和服务安全风险

| 端口    | 服务      | 安全风险       | 防御措施          |
| ----- | ------- | ---------- | ------------- |
| 21    | FTP     | 明文传输、弱口令   | 使用SFTP、禁用匿名访问 |
| 22    | SSH     | 暴力破解、弱密钥   | 更改默认端口、使用密钥认证 |
| 23    | Telnet  | 明文传输       | 禁用，使用SSH替代    |
| 25    | SMTP    | 邮件中继、垃圾邮件  | 配置认证、限制中继     |
| 80    | HTTP    | 明文传输、XSS   | 使用HTTPS、输入验证  |
| 443   | HTTPS   | SSL/TLS漏洞  | 及时更新证书、使用强加密  |
| 3389  | RDP     | 暴力破解、远程控制  | 限制访问IP、使用强密码  |
| 3306  | MySQL   | 弱口令、未授权访问  | 限制监听地址、使用强密码  |
| 6379  | Redis   | 未授权访问、配置错误 | 设置密码、限制访问IP   |
| 27017 | MongoDB | 未授权访问      | 设置密码、限制访问IP   |

## 实操项目

### 项目1：DVWA本地搭建与漏洞练习

**目标**：通过DVWA（Damn Vulnerable Web Application）练习常见Web漏洞

**步骤**：

1. **搭建DVWA环境**：
   - 使用Docker快速搭建：`docker run -d -p 8080:80 vulnerables/web-dvwa`
   - 访问 <http://localhost:8080，默认账号：admin/password>
2. **完成low级别所有漏洞**：
   - SQL注入：使用简单的SQL语句获取数据库信息
   - XSS：注入脚本获取cookie
   - CSRF：构造恶意请求
   - 文件上传：上传webshell
   - 命令注入：执行系统命令
3. **记录练习过程**：
   - 每个漏洞的原理
   - 攻击步骤
   - 防御方法
