## 学习内容

### 3.1 Linux系统日志

#### 常见日志文件
- **/var/log/messages**：系统核心日志，记录系统启动、运行状态等
- **/var/log/auth.log**（Debian/Ubuntu）或 **/var/log/secure**（CentOS/RHEL）：认证日志，记录登录、sudo等操作
- **/var/log/nginx/access.log**：Nginx访问日志
- **/var/log/nginx/error.log**：Nginx错误日志
- **/var/log/apache2/access.log**：Apache访问日志
- **/var/log/apache2/error.log**：Apache错误日志
- **/var/log/secure**：安全相关日志
- **/var/log/cron**：定时任务日志
- **/var/log/maillog**：邮件服务日志

#### 日志格式分析
- **系统日志格式**：通常包含时间戳、主机名、进程名、PID、日志内容
- **Web访问日志格式**：通常包含IP地址、时间、请求方法、URL、状态码、响应大小等

### 3.2 日志分析工具

#### 命令行工具
- **grep**：文本搜索
  - 基本用法：`grep "error" /var/log/messages`
  - 高级用法：`grep -i "failed" /var/log/auth.log | grep "ssh"`

- **awk**：文本处理
  - 基本用法：`awk '{print $1, $4, $7}' /var/log/nginx/access.log`
  - 高级用法：`awk '$9 == 404 {print $1, $7}' /var/log/nginx/access.log`

- **sed**：文本替换
  - 基本用法：`sed 's/old/new/g' /var/log/messages`
  - 高级用法：`sed -n '/2023-04-01/p' /var/log/messages`

- **tail**：查看日志尾部
  - 基本用法：`tail -f /var/log/messages`（实时查看）
  - 高级用法：`tail -n 100 /var/log/messages`（查看最后100行）

- **head**：查看日志头部
  - 基本用法：`head -n 50 /var/log/messages`（查看前50行）

- **sort**：排序
  - 基本用法：`sort /var/log/messages`
  - 高级用法：`sort -k 3 /var/log/messages`（按第3列排序）

- **uniq**：去重
  - 基本用法：`uniq /var/log/messages`
  - 高级用法：`uniq -c /var/log/messages`（统计重复次数）

#### 日志分析平台
- **ELK Stack**：
  - Elasticsearch：存储和索引日志
  - Logstash：收集和处理日志
  - Kibana：可视化和查询日志

- **Graylog**：集中式日志管理平台
- **Splunk**：企业级日志分析平台
- **Loki**：轻量级日志聚合系统

### 3.3 防火墙日志分析

#### 常见防火墙日志
- **iptables日志**：通常位于 `/var/log/kern.log` 或 `/var/log/messages`
- **Firewalld日志**：通常位于 `/var/log/firewalld`
- **硬件防火墙日志**：通常通过Syslog发送到日志服务器

#### 防火墙日志格式
- 通常包含时间戳、动作（允许/拒绝）、源IP、目标IP、协议、端口等信息

### 3.4 日志分析技巧

#### 异常检测
- **登录异常**：多次登录失败、非常规时间登录、来自异常IP的登录
- **访问异常**：大量404错误、异常的请求方法、异常的URL路径
- **系统异常**：服务启动/停止、进程崩溃、资源使用异常

#### 安全事件识别
- **暴力破解**：多次失败的登录尝试
- **扫描行为**：对多个端口的扫描请求
- **SQL注入**：包含SQL语句的请求
- **XSS攻击**：包含脚本的请求
- **命令注入**：包含系统命令的请求

## 实操项目

### 项目3：日志分析实战

**目标**：分析公司Web服务器日志，识别异常访问

**步骤**：
1. **日志收集**：
   - 获取Web服务器（Nginx/Apache）的访问日志
   - 确保日志格式正确

2. **基本分析**：
   - 使用grep/awk分析日志
   - 统计访问量最高的IP地址
   - 统计访问量最高的URL
   - 统计返回404、500等错误的请求

3. **异常识别**：
   - 识别扫描行为（如对多个不存在的路径的请求）
   - 识别暴力破解尝试（如多次失败的登录请求）
   - 识别SQL注入尝试（如包含SQL语句的请求）
   - 识别XSS攻击尝试（如包含脚本的请求）

4. **脚本编写**：
   - 编写Python或Shell脚本，自动分析日志
   - 脚本功能：
     - 统计访问量
     - 识别异常IP
     - 识别异常URL
     - 生成分析报告

5. **报告生成**：
   - 整理分析结果
   - 提出安全建议
