# 2. 安全工具基础（4周）

## 学习目标
- 掌握常见安全扫描工具的使用
- 能够进行基础的漏洞扫描和分析
- 熟悉网络资产发现和服务识别
- 了解Web应用安全测试工具的使用

## 学习内容

### 2.1 端口扫描：Nmap

#### 基础命令
- **基本扫描**：`nmap 192.168.1.1`
- **指定端口**：`nmap -p 21-80 192.168.1.1`
- **服务识别**：`nmap -sV 192.168.1.1`
- **操作系统识别**：`nmap -O 192.168.1.1`
- **综合扫描**：`nmap -A 192.168.1.1`
- **扫描网段**：`nmap 192.168.1.0/24`

#### 高级用法
- **UDP扫描**：`nmap -sU 192.168.1.1`
- **SYN扫描**：`nmap -sS 192.168.1.1`（半开放扫描，速度快）
- **隐蔽扫描**：`nmap -sN -sF -sX 192.168.1.1`（无状态扫描，更隐蔽）
- **脚本扫描**：`nmap --script vuln 192.168.1.1`（使用漏洞检测脚本）

### 2.2 漏洞扫描：Nessus

#### 安装与配置
- **下载安装**：访问 https://www.tenable.com/downloads/nessus 下载
- **激活**：获取激活码并激活
- **插件更新**：首次启动时会自动更新插件

#### 扫描配置
1. **创建扫描**：
   - 点击"New Scan"
   - 选择扫描模板（如"Basic Network Scan"）
   - 输入目标IP或网段
   - 配置扫描选项

2. **扫描类型**：
   - **基础网络扫描**：适用于一般网络
   - **Web应用扫描**：针对Web应用
   - **合规性扫描**：检查合规性
   - **高级扫描**：自定义扫描选项

#### 报告解读
- **漏洞等级**：Critical（严重）、High（高）、Medium（中）、Low（低）
- **漏洞详情**：漏洞描述、CVSS评分、修复建议
- **报告导出**：支持PDF、HTML、CSV等格式

### 2.3 Web扫描：Goby

#### 安装与配置
- **下载**：访问 https://gobysec.net/ 下载
- **安装**：按照提示完成安装
- **更新**：定期更新漏洞库

#### 使用方法
1. **资产扫描**：
   - 输入目标IP或域名
   - 选择扫描模式（快速/全端口）
   - 开始扫描

2. **漏洞检测**：
   - 扫描完成后，点击"漏洞检测"
   - 选择漏洞类型
   - 开始检测

3. **报告生成**：
   - 扫描完成后，点击"报告"
   - 选择报告格式
   - 导出报告

### 2.4 Web扫描：AWVS

#### 安装与配置
- **下载**：访问 https://www.acunetix.com/ 下载
- **安装**：按照提示完成安装
- **激活**：使用许可证激活

#### 使用方法
1. **添加目标**：
   - 点击"Add Target"
   - 输入目标URL
   - 配置扫描选项

2. **开始扫描**：
   - 选择扫描类型（Full Scan/Quick Scan）
   - 点击"Scan"

3. **报告查看**：
   - 扫描完成后，查看漏洞详情
   - 导出报告

### 2.5 抓包分析：Burp Suite

#### 安装与配置
- **下载**：访问 https://portswigger.net/burp 下载
- **安装**：按照提示完成安装
- **配置代理**：
  - 浏览器设置代理：127.0.0.1:8080
  - 导入CA证书

#### 基础功能
1. **Proxy**：
   - 拦截和修改HTTP请求/响应
   - 查看请求详情

2. **Scanner**：
   - 自动扫描Web应用漏洞
   - 查看扫描结果

3. **Intruder**：
   - 暴力破解
   - 模糊测试

4. **Repeater**：
   - 重复发送请求
   - 修改请求参数

## 实操项目

### 项目1：公司内网资产梳理

**目标**：使用Nmap扫描公司内网，识别资产和潜在风险

**步骤**：
1. **网络规划**：
   - 确定需要扫描的网段
   - 制定扫描计划

2. **执行扫描**：
   - 使用Nmap扫描内网段：`nmap -A 192.168.1.0/24`
   - 记录开放端口和服务

3. **资产整理**：
   - 创建资产清单，包含：IP地址、主机名、开放端口、运行服务、版本信息
   - 识别潜在风险点（如弱口令服务、不必要的开放端口）

4. **报告生成**：
   - 整理扫描结果
   - 提出安全建议

### 项目2：漏洞扫描实践

**目标**：使用Nessus或OpenVAS对测试环境进行漏洞扫描

**步骤**：
1. **环境准备**：
   - 选择测试环境（如虚拟机）
   - 确保测试环境可访问

2. **执行扫描**：
   - 使用Nessus创建扫描任务
   - 配置扫描选项
   - 开始扫描

3. **报告解读**：
   - 分析扫描报告
   - 理解漏洞等级和影响
   - 提出修复建议

4. **验证修复**：
   - 实施修复措施
   - 重新扫描验证修复效果
