# 网络安全学习规划

> 目标：从网络运维转型安全运维，最终成为电力行业安全工程师
> 预计周期：12-18个月
> 起始时间：2026年4月

---

## 第一阶段：安全基础（0-3个月）

### 目标
- 建立安全基础知识体系
- 掌握常见安全工具使用
- 能够进行基础的安全巡检

### 学习内容

#### 1. 安全基础概念（2周）
- [ ] CIA三要素（机密性、完整性、可用性）
- [ ] 常见攻击类型：SQL注入、XSS、CSRF、文件上传、命令注入
- [ ] OWASP Top 10漏洞原理
- [ ] 常见端口和服务安全风险（21/22/23/25/80/443/3389等）

**推荐资源：**
- 书：《白帽子讲Web安全》前5章
- 视频：B站「泷羽sec」Web安全入门系列
- 练习：DVWA本地搭建，完成low级别所有漏洞

#### 2. 安全工具基础（4周）
- [ ] 漏洞扫描：Nessus安装使用、扫描报告解读
- [ ] 端口扫描：Nmap基础命令、服务识别
- [ ] Web扫描：Goby、AWVS基础使用
- [ ] 抓包分析：Burp Suite基础操作

**实操项目：**
```
项目1：公司内网资产梳理
- 用Nmap扫描公司内网段，识别开放端口
- 整理资产清单（IP、端口、服务）
- 识别潜在风险点（弱口令服务、不必要的端口）

项目2：漏洞扫描实践
- 用Nessus或OpenVAS扫描测试环境
- 解读扫描报告，理解漏洞等级
- 提出修复建议
```

#### 3. 日志分析基础（3周）
- [ ] Linux系统日志：/var/log/下常见日志文件
- [ ] Web日志：Nginx/Apache访问日志、错误日志
- [ ] 防火墙日志分析
- [ ] 日志分析工具：grep/awk/sed基础、ELK概念

**实操项目：**
```
项目3：日志分析实战
- 分析公司Web服务器日志
- 识别异常访问（扫描行为、暴力破解）
- 编写简单的日志分析脚本（Python或Shell）
```

#### 4. 网络安全基础（1周）
- [ ] 防火墙原理与配置
- [ ] IDS/IPS概念
- [ ] VPN安全
- [ ] 网络分段与隔离

**结合工作：**
- 梳理公司现有网络架构和安全设备
- 了解防火墙规则配置逻辑

## 第二阶段：安全运维实践（3-9个月）

### 目标
- 独立承担安全运维工作
- 考取CISP或Security+证书
- 参与公司安全合规工作

### 学习内容

#### 1. 安全设备运维（4周）
- [ ] 防火墙配置与管理（深信服、华为、山石网科）
- [ ] WAF配置与规则调优
- [ ] 堡垒机使用与审计
- [ ] IDS/IPS告警分析与处理

**实操项目：**
```
项目4：安全设备优化
- 检查防火墙规则，清理无用规则
- 配置WAF基础防护规则
- 分析堡垒机审计日志，发现异常操作
```

#### 2. 漏洞管理（3周）
- [ ] 漏洞生命周期管理
- [ ] CVE漏洞跟踪
- [ ] 补丁管理流程
- [ ] 漏洞修复验证

**实操项目：**
```
项目5：漏洞管理流程建立
- 建立漏洞扫描计划（每月/每季度）
- 创建漏洞跟踪表
- 推动漏洞修复
- 复验修复效果
```

#### 3. 安全加固（4周）
- [ ] Linux服务器安全加固
  - 账号安全（密码策略、sudo配置）
  - SSH加固（禁用root登录、密钥认证）
  - 服务最小化
  - 文件权限
- [ ] 网络设备安全加固
- [ ] 中间件安全配置（Nginx、Tomcat）

**实操项目：**
```
项目6：服务器安全加固
- 选择测试服务器进行加固
- 编写加固Checklist
- 形成加固文档
```

#### 4. 应急响应基础（3周）
- [ ] 应急响应流程（PDCERF模型）
- [ ] Webshell查杀
- [ ] 入侵痕迹分析
- [ ] 日志溯源

**实操项目：**
```
项目7：应急响应演练
- 模拟Webshell入侵场景
- 练习排查思路
- 编写应急响应报告
```

#### 5. 证书备考（穿插进行）

**Security+ 备考计划（推荐先考）：**
| 周次 | 学习内容 |
|------|----------|
| 第1-2周 | 攻击、威胁和漏洞（24%） |
| 第3-4周 | 架构和设计（21%） |
| 第5-6周 | 实施（25%） |
| 第7-8周 | 运营和应急响应（16%） |
| 第9-10周 | 治理、风险和合规（14%） |
| 第11-12周 | 模拟题+错题复习 |

**考试信息：**
- 考试费：约370美元（人民币2700元左右）
- 题型：选择题，90题，90分钟
- 及格分：750/900
- 有效期：3年

**CISP 备考计划（国内认可度高）：**
- 建议报培训班（培训费+考试费约1万）
- 培训周期：5天集中培训
- 考试题型：选择题，100题
- 及格分：70分

## 第三阶段：安全工程师进阶（9-18个月）

### 目标
- 掌握渗透测试基础
- 考取CISP-PTE或OSCP
- 成为电力行业安全专家

### 学习内容

#### 1. 渗透测试基础（8周）

**Web渗透（4周）：**
- [ ] 信息收集：子域名、端口、目录、指纹识别
- [ ] SQL注入：手工注入、工具使用、绕过技巧
- [ ] XSS：反射型、存储型、DOM型
- [ ] 文件上传漏洞：绕过技巧
- [ ] 文件包含漏洞
- [ ] SSRF/CSRF
- [ ] 逻辑漏洞

**工具学习：**
- Burp Suite进阶（Intruder、Repeater、插件）
- SQLMap
- Nmap脚本
- 其他：Dirsearch、Whatweb、Wpscan

**内网渗透（4周）：**
- [ ] 内网信息收集
- [ ] 横向移动
- [ ] 权限提升（Linux/Windows）
- [ ] 域渗透基础
- [ ] 隧道技术

**工具学习：**
- Metasploit基础
- Cobalt Strike概念
- 内网扫描工具

**练习平台：**
- DVWA（本地）
- Pikachu（本地）
- HackTheBox（在线，推荐Starting Point系列）
- TryHackMe（在线，推荐Complete Beginner路径）
- VulnHub（本地虚拟机）

**实操计划：**
```
每周目标：
- 完成1-2个靶机
- 记录渗透思路
- 总结学到的新技巧
```

#### 2. Python安全脚本（4周）
- [ ] Python基础语法复习
- [ ] 端口扫描脚本
- [ ] 目录扫描脚本
- [ ] 日志分析脚本
- [ ] 资产收集脚本
- [ ] 简单POC编写

**实操项目：**
```
项目8：自动化工具开发
- 编写一个端口扫描器
- 编写一个目录扫描器
- 编写日志分析脚本
```

#### 3. 电力行业安全（持续学习）
- [ ] 电力行业安全规范学习
- [ ] 工控安全基础（SCADA、PLC、DCS）
- [ ] 等保2.0电力行业要求
- [ ] 关键信息基础设施保护条例

**推荐资源：**
- 《工业控制系统安全》
- 国家能源局安全相关文件
- 等保2.0标准（GB/T 22239-2019）

#### 4. 进阶证书

**CISP-PTE（渗透测试工程师）：**
- 国内渗透测试认证
- 培训+考试费约1.2万
- 考试形式：实操
- 适合在国内发展

**OSCP（ Offensive Security Certified Professional）：**
- 国际认可度高
- 考试费约1000美元
- 考试形式：24小时实操
- 难度较大，含金量高

## 学习方法建议

### 1. 输入与输出结合
- 看书/视频占40%
- 动手实践占60%
- 每学一个知识点，必须动手验证

### 2. 项目驱动
- 不要只学理论，要结合公司实际情况
- 每个阶段至少完成一个实操项目
- 项目成果可以汇报给领导，展示价值

### 3. 记录与复盘
- 建立学习笔记（Markdown格式）
- 记录每个靶机的渗透思路
- 定期复盘，总结经验

### 4. 社区参与
- 加入安全交流群
- 关注安全大佬博客/公众号
- 参与CTF比赛（可选）

### 5. 结合工作
- 在日常运维中发现安全问题
- 把学到的工具用在公司环境（测试环境）
- 主动承担安全相关工作

## 注意事项

1. **循序渐进**：不要一开始就学渗透测试，基础很重要
2. **合法合规**：所有渗透测试练习必须在授权环境进行
3. **持续学习**：安全领域更新快，保持学习习惯
4. **理论结合实践**：光看不练是学不会安全的
5. **保护公司信息**：学习过程中不要泄露公司敏感信息
